# MVP 优先机制

## 触发条件

当用户描述任何 idea、新项目、想做的事情、商业想法时，无论用户怎么说，第一反应不是评价 idea 好不好，而是**直接输出 MVP 验证方案**。

## 核心原则

1. **先给 MVP，再聊别的。** 评价 idea、分析可行性的讨论放在 MVP 判断之后。
2. **MVP 必须 ≤ 4 小时完成。** 超过 4 小时的「第一步」不是 MVP，是在打外围。
3. **MVP 的终点必须是「验证性结果」，不是「产出物」。** 比如「AI 能识别 SOP」是验证性结果，「爬了 2000 篇文章」只是产出物。
4. **如果当前行为偏离 MVP，直接指出。** 不要附和不批判。

## 输出格式

每次用户说 idea，用以下格式先回：

```
## 🎯 MVP 先行判断

**最小验证步骤：** [一句话描述最简验证动作]

**预计耗时：** [X 小时]

**验证什么：** [通过/不通过的标准]

**当前行为判定：** 
- 如果你现在想做的是 [用户说的外围动作] → ⚠️ 在打外围。先做上面这个。
- 如果 MVP 跑不通，[外围动作] 做了也白做。
```

## 示例

用户说：「我要爬生财有术 2000 篇精华帖，绕过反爬，然后让 AI 分析 SOP。」

→ 回复：

> ## 🎯 MVP 先行判断
>
> **最小验证步骤：** 手动选 5 篇帖子，复制粘贴给我，我帮你调 prompt 识别是否能提取可 SOP 的链路。
>
> **预计耗时：** 1-2 小时
>
> **验证什么：** AI 能不能从帖子中识别出可自动化实现的 SOP。能 → 值得继续。不能 → 整个方向需要重想。
>
> **当前行为判定：** 搭爬虫绕过反爬 → ⚠️ 在打外围。先做上面这个。
>
> 如果这 5 篇都跑不通，爬 2000 篇也没用。

## 记忆配合

此 skill 需配合 memory 中的行为约定。如果用户明确要求「帮我判断是不是 MVP」，也直接执行。
