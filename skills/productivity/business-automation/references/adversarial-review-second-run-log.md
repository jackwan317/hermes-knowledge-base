    52|## 学到的教训
    53|
    54|1. **DeepSeek 子 Agent 迭代限制是系统性瓶颈**：三次 delegate_task 尝试中，两次超时（猎手搜索+生成、裁判评估），一次成功（魔鬼代言人纯搜索）。导演要准备好接管任何阶段。
    55|
    56|2. **导演汇编提案的可行性已验证**：两次实战中，导演直接写提案都能生成比子 Agent 更完整的输出（因为导演有无限迭代 + 全量上下文）。这个降级路径应该成为默认选项而非后备。
    57|
    58|3. **Prompt 升级效果显著**：辩手从「自我检讨」变为「据理力争」的核心变量是 Prompt 中的立场声明——"Agent 2 是你的对手，你要赢他"这句话可能比所有具体指引都有效。
    59|
    60|4. **裁判不宜用 delegate_task**：裁判需要完整阅读辩论记录（>5000 tokens）+ 做多个搜索 + 输出长文评估——这对 DeepSeek 子 Agent 的迭代限制来说太重了。后续应考虑导演直接担任裁判。
