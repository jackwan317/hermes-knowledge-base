## Q1 · 市场规模与增长潜力

提案中的 TAM/SAM/SOM 全部标注「待验证」，唯一相对可靠的数据来自 Reddit 社区规模统计（r/passive_income 1.8M, r/sidehustle 2.3M）。但社区成员数 ≠ 付费用户数。r/passive_income 的热门帖子是做问卷调查赚 $5/小时的零工，这些人的付费意愿和你的 $29-49/月定价之间存在巨大鸿沟。

**问题**：你能找到任何一个已经验证的「按月付费的 AI 自动赚钱工具」的市场数据点吗？哪怕是间接参照物——有没有类似的「AI 替你做事赚钱」的 SaaS 产品做到了 $100K+ ARR？如果找不到，你是怎么排除「这个市场根本不存在」的可能性的？

## Q2 · 竞争格局与差异化壁垒

提案说「消费级 AI 自主赚钱领域基本真空」。但同时也承认 AutoGPT (160k stars) 从消费级转型 B2B，AgentGPT 转型企业数据管道。这两个产品的转型轨迹暗示：消费级市场可能不是「无人做」而是「做了但养不活」。

**问题**：AutoGPT 和 AgentGPT 在消费级上失败的具体原因是什么？是留存率低？是付费率低？是 API 成本吃掉了利润？如果你不清楚他们失败的原因，你凭什么认为 AgentMint 能避开？你的差异化不是「更好的执行」——因为竞品失败未必是执行问题，可能是商业模式先天性缺陷。

## Q3 · 技术可行性与实现路径

提案的核心闭环是「AI 自动发现 → 验证 → 执行 → 收款」。但在「未解决的问题」部分，你承认了三个硬伤：验证没反馈、支付被标记、内容被处罚。这三个问题不是「待优化」，而是致命性未解决——任何一个堵死，整个闭环就断了。

**问题**：你能在不依赖「平台仁慈」的前提下，给出任何一个闭环的端到端技术 demo 思路吗？具体地说：如果 Stripe 对 AI 账户封禁、Google 对 AI 内容降权、Gumroad 要求人工审核——你怎么让 Agent 收到第一笔真金白银？请给出具体的替代收款路径，不是「分流」或「缓解」，是可靠路径。

## Q4 · 商业模式与盈利能力

单位经济模型中，每用户月 API 成本 $5-15，定价 $29-49。但 Agent 要持续跑 (搜索+执行)，API 消耗不会低。加上「用户输入偏好 → Agent 搜索 → 测试可行性 → 执行」这条链，假设每个用户每天 Agent 运行 3 次，每次 10 轮 LLM 调用，日消耗轻松 $1-2。月 API 成本可能接近 $30-60——已经超过订阅收入。

**问题**：你计算过单用户的真实 API 消耗量吗？Agent 每天需要跑多少次「搜索金钱机会」的循环？每次循环会触发多少 LLM 调用？给出一个带具体假设的计算，证明 $29-49/月的定价在 API 成本上可以支撑。

## Q5 · 用户需求真实性与痛点强度

提案列举的痛点——「发现难、执行难、持续难」——是 2015 年就在说的老问题。Reddit 上大量帖子问「我该做什么」不等于「我愿意付费让 AI 替我做」。用户真正的行为是：看到「AI 自动赚钱」的广告 → 想试 → 搜免费替代 → 发现 ChatGPT 能提供类似建议 (免费) → 放弃付费。

**问题**：你打算如何区分「用户说想要」和「用户真会付钱」？你的 landing page 验证方法 (转化率 >3%、50 个 email signup) 是标准做法，但如果转化率只有 0.5% (这在未验证品类中很常见)，你的判断是什么？退出还是 pivot？

## Q6 · 团队能力与资源匹配度

提案中「大哥的独特优势」完全基于 GitHub 和飞书记录的推断，你自己也承认「未进行正式的团队能力评估」。列出的四个优势中，Hermes Agent 架构经验、飞书工作流、技术栈匹配都是间接相关的，不是直接竞争力。唯一直接相关的「对睡后收入商业模式的理解」没有任何具体案例支撑。

**问题**：如果大哥不做这个项目，他现在最有实力做出的是什么？（给出一个基于他实际技术栈和能力的具体替代方向）如果你连这个都答不上来，你怎么确信 AgentMint 是他最匹配的项目而不是你用「AI 赚钱」这个性感关键词生搬硬套出来的？

## Q7 · 时间窗口与市场时机

提案说「先发优势窗口 6-12 个月」。但你引用的竞品 AutoGPT 早在 2023 年就证明了需求存在，两年过去了，没有任何消费级「AI 自己赚钱」的产品跑出来。为什么窗口突然在 2026 年打开了？是因为技术变了还是因为没人想到？如果两年前没人做成，为什么现在有了 browser-use 就突然可行了——browser-use 2024 年就有了。

**问题**：给出至少一个 2025-2026 年间发生的具体技术突破或行业事件，证明「现在比 2023 年更可行」。不要泛泛说「API 降价了」——GPT-4o 比 GPT-4 便宜 50% 但 AutoGPT 失败的原因从来不是 API 太贵。

## Q8 · 平台依赖、政策与合规风险

风险表里列了 5 项，其中 Stripe、Reddit、Google 三个是致命级。你给 Stripe 的应对是「人机协作」「Lemon Squeezy 分流」，给 Google 的是「人工编辑混合」。但你没回答最核心的问题：这些综合起来意味着什么？

**问题**：如果同时发生以下两件事——(a) Stripe 收紧 AI 自动化政策 (b) Google 更新算法打击 AI 生成内容站——你的产品还剩什么？你能在只剩 Gumroad 和直接法币收款的情况下，维持哪怕 30% 的收入吗？请具体回答，不要回避。

## Q9 · 获客路径与规模化能力

获客计划只有一个短句：「Product Hunt launch + Reddit + SEO 长尾」。但你自己的竞品分析中指出：Reddit 内容站被 API 政策杀死 (GummySearch)，且大量低质量 AI 插件在 Chrome Web Store 生命周期 <6 个月。Reddit 对 self-promotion 极其敏感——如果帖子被 r/sidehustle 版主标记为广告，你的获客路径就断了。

**问题**：在 Reddit 和 SEO 长尾都不可靠的情况下（前者可能被删帖，后者需要 6-12 个月才能见效），你前 3 个月的具体获客计划是什么？给出 CAC <$100 的计算依据——不是行业平均值，是与你的产品和定价直接相关的计算。

## Q10 · 关键假设与最大失败风险

三个关键假设验证成本加起来 $1,200，时间 5 周。但验证方法有逻辑漏洞：假设 1 的「手动搭建 MVP」不等同于「Agent 自动跑通」，因为人能做的不代表 Agent 能可靠复制。假设 2 的「测试账户运行 1 个月」——Stripe 的风控可能对真实用户规模（而不是 1 个测试账户）触发封禁。假设 3 的「landing page 转化率 >3%」——对完全没有品类认知的新概念，这个阈值过于乐观（行业基准：完全新品类 landing page 转化率通常在 1-2%）。

**问题**：三个假设中，你认为哪一个最可能被证伪？如果三个假设中两个被推翻，你是 pivot 到路径 B (人机协作) 还是路径 C (B2B)，还是直接关停？给出一个明确的「go/no-go」决策框架，不是模棱两可的「看情况」。
